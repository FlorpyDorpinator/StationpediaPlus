                STATIONPEDIA ASCENDED - UI ENHANCEMENT TEST CHECKLIST

Test the following features after building and loading the mod in Stationeers.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

    - Previous: 28x28 pixels
    - New: 32x32 pixels

--------------------------------------------------------------------------------
2. HOME PAGE BUTTON LAYOUT
--------------------------------------------------------------------------------
[ ] Open Stationpedia home page

NEW CUSTOM BUTTONS (Above Guides/Universe):
[ ] Two new buttons should appear ABOVE the Guides and Universe buttons
[ ] "Stationeers Survival Manual" button should appear
[ ] "Game Mechanics" button should appear  
[ ] Both buttons should use the SAME font, text size, and button style as Guides/Universe
[ ] Buttons should be in two-column layout matching Guides/Universe format
[ ] Button colors should match vanilla Stationpedia style

EXISTING BUTTONS:
[ ] Guides button should still work (NOT redirect to Game Mechanics)
[ ] Universe button should still work
[ ] Category buttons (Reagents, Gases, Ingots, etc.) should work normally
[ ] NO buttons should overlap or be in wrong positions

--------------------------------------------------------------------------------
3. SURVIVAL MANUAL PAGE
--------------------------------------------------------------------------------
[ ] Click "Stationeers Survival Manual" button
[ ] Page should open without errors

PART STRUCTURE (3 Major Collapsibles):
[ ] Part 1 header should appear as a MAJOR collapsible section
[ ] Part 2 header should appear as a MAJOR collapsible section  
[ ] Part 3 header should appear as a MAJOR collapsible section
[ ] All three Parts should DEFAULT TO OPEN when page first loads
[ ] Parts should save their collapsed state if player closes them
[ ] NO "Operational Details" wrapper should exist - Parts are top-level

TABLE OF CONTENTS (Per Part):
[ ] Each Part should have its own Table of Contents box at the top
[ ] "Table Of Contents" title should be CENTERED in the box
[ ] ToC should display up to 3 columns if content is long enough
[ ] Section titles in ToC should be UNDERLINED AND BOLDED (vanilla mode)
[ ] Subsection entries should have dashes, NOT underlined
[ ] ToC box should have its own background, separate from content
[ ] ToC links should be CLICKABLE and scroll to the section

CONTENT SECTIONS:
[ ] Sections within each Part should be in their own collapsible boxes
[ ] Section titles should display correctly
[ ] Content text should render properly
[ ] Vanilla syntax should work: {LINK:}, {THING:}, {HEADER:}

NAVIGATION:
[ ] ToC links should expand parent sections if needed and scroll to target
[ ] Back button should return to home page
[ ] Manual should be accessible from both vanilla and ascended modes

--------------------------------------------------------------------------------
4. GAME MECHANICS PAGE
--------------------------------------------------------------------------------
[ ] Click "Game Mechanics" button
[ ] Page should open without errors
[ ] Page should show "Game Mechanics" title
[ ] Currently empty is OK - content not yet added
