================================================================================
STATIONPEDIA ASCENDED v0.3.0 - LLM INSTRUCTIONS FOR WRITING descriptions.json
================================================================================

You are helping write device documentation for the Stationeers game mod 
"Stationpedia Ascended". This mod enhances the in-game encyclopedia with 
custom tooltips, collapsible sections, table of contents, images, and videos.

================================================================================
FILE STRUCTURE
================================================================================

The descriptions.json file has this top-level structure:

{
  "version": "1.0",
  "devices": [ ... array of device entries ... ],
  "genericDescriptions": { ... global fallback descriptions ... }
}

================================================================================
DEVICE ENTRY FORMAT (Complete Reference)
================================================================================

{
  "deviceKey": "ThingStructureDeviceName",      // REQUIRED - exact prefab name
  "displayName": "Human Readable Name",          // Optional - for documentation

  // === PAGE DESCRIPTION OPTIONS (pick ONE or combine append/prepend) ===
  "pageDescription": "Replaces entire description",
  "pageDescriptionAppend": "Added to end of existing description",
  "pageDescriptionPrepend": "Added to beginning of existing description",

  // === TABLE OF CONTENTS (TOC) ===
  "generateToc": true,                           // Enable TOC panel at top
  "tocTitle": "Quick Navigation",                // Custom TOC title (default: "Contents")

  // === OPERATIONAL DETAILS (MUST BE AN ARRAY) ===
  "operationalDetails": [ ... ],                 // Main content sections
  "operationalDetailsTitleColor": "#FF7A18",     // Title color (hex)
  "operationalDetailsBackgroundColor": "#1A2B3C", // Section background (hex)

  // === TOOLTIP DESCRIPTIONS (Objects with keys) ===
  "logicDescriptions": { ... },
  "modeDescriptions": { ... },
  "slotDescriptions": { ... },
  "versionDescriptions": { ... },
  "memoryDescriptions": { ... }
}

================================================================================
OPERATIONALDETAILS - ALL SUPPORTED PROPERTIES
================================================================================

Each item in the operationalDetails array can have these properties:

BASIC CONTENT:
  "title"           - (string) Section heading
  "description"     - (string) Main text content with TMP rich text support
  "items"           - (array of strings) Bullet list (rendered with • prefix)
  "steps"           - (array of strings) Numbered list (rendered as 1. 2. 3.)
  "children"        - (array) Nested OperationalDetail objects for sub-sections

COLLAPSIBLE SECTIONS:
  "collapsible"     - (boolean) If true, renders as expandable/collapsible section
                      Default: false (renders as inline text)

TABLE OF CONTENTS:
  "tocId"           - (string) Unique ID for TOC linking. When set:
                      - Section appears in the Table of Contents
                      - Clicking TOC link scrolls to this section
                      - Works on both collapsible and non-collapsible sections

MEDIA:
  "imageFile"       - (string) Image filename in mod/images/ folder
                      Supports: PNG, JPG, GIF (first frame only)
  "videoFile"       - (string) Video filename in mod/images/ folder
                      Supports: MP4. Renders as embedded video player
  "youtubeUrl"      - (string) Full YouTube URL (e.g., "https://www.youtube.com/watch?v=...")
                      Renders as clickable link that opens in system browser
  "youtubeLabel"    - (string) Custom label for YouTube link
                      Default: "Watch on YouTube"

STYLING:
  "backgroundColor" - (string) Custom hex color for this section's background
                      Format: "#RRGGBB" (e.g., "#1A2B3C")

================================================================================
COLLAPSIBLE VS INLINE CONTENT
================================================================================

COLLAPSIBLE (collapsible: true):
  - Renders as expandable StationpediaCategory with header bar
  - User can click to expand/collapse
  - Supports nested children that also render as collapsibles
  - Best for: Long content, optional reading, nested documentation

INLINE (collapsible: false or not set):
  - Renders as regular text directly in the parent section
  - Always visible, cannot be collapsed
  - Best for: Short content, important notes, simple lists

Example:
{
  "title": "This is collapsible",
  "collapsible": true,
  "description": "User must click to see this content"
}

{
  "title": "This is inline",
  "description": "This content is always visible"
}

================================================================================
TABLE OF CONTENTS (TOC) SYSTEM
================================================================================

To enable TOC for a device:

1. Set "generateToc": true on the device entry
2. Add "tocId" to each section you want in the TOC

The TOC panel:
  - Appears at the top of Operational Details
  - Shows clickable links to each tocId section
  - Automatically uses multi-column layout (max 8 rows per column)
  - Clicking a link scrolls to and expands that section

Example:
{
  "deviceKey": "ThingRobot",
  "generateToc": true,
  "tocTitle": "AIMeE Guide",
  "operationalDetails": [
    {
      "title": "Overview",
      "tocId": "overview",
      "collapsible": true,
      "description": "Introduction to AIMeE..."
    },
    {
      "title": "Modes",
      "tocId": "modes",
      "collapsible": true,
      "children": [
        {
          "title": "Follow Mode",
          "tocId": "mode-follow",
          "collapsible": true,
          "description": "AIMeE follows the player..."
        }
      ]
    }
  ]
}

================================================================================
TEXTMESHPRO RICH TEXT FORMATTING
================================================================================

The mod uses TextMeshPro for rendering. Supported tags:

BASIC:
  <b>bold</b>
  <i>italic</i>
  <u>underline</u>
  <s>strikethrough</s>

COLORS:
  <color=#FF0000>hex color</color>
  <color=red>named color</color>

SIZE:
  <size=150%>bigger</size>
  <size=80%>smaller</size>
  <size=24>24pt</size>

ALIGNMENT:
  <align=center>centered</align>
  <align=right>right-aligned</align>

INDENTATION:
  <indent=10%>indented text</indent>

SUPER/SUBSCRIPT:
  H<sub>2</sub>O  →  H₂O
  x<sup>2</sup>   →  x²

OTHER:
  <nobr>no line break</nobr>
  <lowercase>LOWERCASE</lowercase>
  <uppercase>uppercase</uppercase>

LINE BREAKS:
  Use \n in JSON strings

================================================================================
VALIDATION CHECKLIST
================================================================================

Before submitting your JSON:

[ ] deviceKey matches exact prefab name (e.g., "ThingStructureFurnace")
[ ] operationalDetails is an ARRAY with square brackets []
[ ] Each collapsible section has "collapsible": true
[ ] tocId values are unique within the device
[ ] generateToc is set to true if using tocId entries
[ ] Rich text tags are properly closed
[ ] Image/video files exist in mod/images/ folder
[ ] YouTube URLs are full URLs (not shortened)
[ ] JSON is valid (no trailing commas, proper brackets)
[ ] Nesting depth is reasonable (2-3 levels max recommended)

================================================================================
COMMON MISTAKES TO AVOID
================================================================================

WRONG - operationalDetails as object:
  "operationalDetails": { "title": "..." }

CORRECT - operationalDetails as array:
  "operationalDetails": [ { "title": "..." } ]

WRONG - Missing collapsible for expandable sections:
  { "title": "Section", "children": [...] }

CORRECT - Explicit collapsible:
  { "title": "Section", "collapsible": true, "children": [...] }

WRONG - tocId without generateToc:
  "operationalDetails": [ { "tocId": "section1" } ]

CORRECT - Enable TOC first:
  "generateToc": true,
  "operationalDetails": [ { "tocId": "section1" } ]
